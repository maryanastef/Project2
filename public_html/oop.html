Principles of Object-Oriented Programming

OOP (Object-Oriented Programming)

In Python, Object-Oriented Programming is a way of programming that uses objects and classes. The main concept of OOP is to connect the data and the functions that work together as a single unit so that no other part of the code can access this data.

An object has two characteristics:

  • Attributes
  • Behavior

The main idea focuses on creating reusable code using OOP principles.

The four principles of object-oriented programming are encapsulation, abstraction, inheritance, polymorphism.

1. Encapsulation:

Encapsulation is when each object keeps its state private, inside a class. Other objects don’t have access to this state, they can only call a list of public functions, known as methods. The object manages its own state using methods, and no other class can access it unless allowed.

A class is a collection of objects and contains attributes and methods. For example, if we apply encapsulation to a calculator example, we encapsulate all calculations into a “Calculation” class. The Calculation class will be able to track different calculations with different attributes like addition, subtraction, multiplication, division.

For more information about encapsulation, please read Encapsulation in Python

2. Abstraction:

Abstraction creates abstract classes, which can be considered as a blueprint for other classes. It allows us to create methods within any child classes. A class that contains one or more abstract methods is called an abstract class.

In Python, Abstraction is used to hide the functionality of the function from the users. It calls for a small set of public methods which any other class can call without knowing how they work.

For more infomation about abstraction, please read Abstract Classes in Python

3. Inheritance:

Inheritance is the capability of one class to derive or inherit the properties from another class. This provides reusability of the code , so you don’t have to write the same code again. Also, it allows us to add more features to a class without modifying it.

It means you create a child class by deriving from another parent class. This way, we form a hierarchy. The child reuses all fields and methods of the parent class (common part) and can implement its own unique part.

In this example, we have a parent class called "Calculator". The class has data attributes to store value x and value y.

[Image from Maryana Stefanyshyn from PyCharm]

Then, we call the class factory method, which is an expression that gets evaluated after your function is defined. The class method can access or modify the class state. It takes cls as its first argument, followed by other arguments and returns a class method for function.

The __init__ method is similar to constructors in other programming languages. It is run as soon as an object of a class is instantiated.

Then, we create a child class Addition, which reuses all fields and methods of the parent class and implements its own unique calculation by adding value x and value y.

[Image from Maryana Stefanyshyn from PyCharm]

4. Polymorphism:

Polymorphism means having “many forms”. In programming, it means having the same function name but different signatures. We can define methods in the child class that have the same name as the methods in the parent class.

Sometimes we want to use a list, which contains a mix of all these classes. Polymorphism allows us to use a class exactly like its parent so there are no mixing types. Each child class keeps its own methods as they are. This happens by defining a parent interface to be reused.

In my calculator example, addition, subtraction, multiplication and division reuse a common interface for their calculations. Having these four calculations inherit the parent Calculator lets me create a list of calculations and treat them like the same type of object.
