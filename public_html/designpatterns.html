Design Patterns

Creational Pattern:

Creational pattern deals with the creation of classes or objects, also known as class instantiation or object instantiation. It is very useful for lowering the level of dependency between classes. Class creation patterns use the instantiation process and object creation patterns use delegations.

Classification of Creational Design Patterns:
  • Factory Method
  • Abstract Factory Method
  • Builder Method
  • Prototype Method
  • Singleton Method

Structural Design Pattern:

Structural Patterns deals with organizing different classes and objects to create new functionality and make it more efficient. This pattern mostly uses inheritance to create all the interfaces.

Classification of Structural Design Patterns:
  • Adapter Method
  • Bridge Method
  • Composite Method
  • Decorator Method
  • Façade Method
  • Proxy Method
  • FlyWeight Method

Behavioral Design Pattern:

Behavioral Pattern deals with identifying communication patterns and responsibilities between objects.

Classification of Behavioral Design Patterns:
  • Chain of Responsibility Method
  • Command Method
  • Iterator Method
  • Mediator Method
  • Memento Method
  • Observer Method
  • State Method
  • Strategy Method
  • Template Method
  • Visitor Method

Example of Structural Pattern:

In this example, the calculator can be considered a façade. The Façade Method is a design pattern that provides a simpler interface to a complex system.

The calculator is a façade on a class called “Calculations”. In our calculations class, all the calculations take place. The calculator wraps the calculations class in a façade and provides an interface to send commands to the calculations class.

[Image from Maryana Stefanyshyn from PyCharm]

By doing this, you can write a program that does a certain set of things you need it to do, instead of thousands of things. Once you put a façade structure around the library to let it do specific things, you wrap the library and only test the functionality that you need to test.

So the façade wraps up the functionality of specific code and provides an easy access to use the code.

Example of Behavioral Pattern:

In my code, I am illustrating the command pattern, which deals with creating a set of commands for the calculator and the user can decide what they want to calculate, instead of us assuming what the user might want.

As you can see, I have created @static method that allows me to pass in as many values as I want and adds the values to a list of numbers.

Then, I created one line code for adding calculation (add_numbers) and subtracting calculation (subtract_numbers). These functions let me send commands to the "Calculations" class for it to do its commands. Addition, Subtraction, Multiplication and Division inherit from calculation and return the result as an instantiated object.

[Image from Maryana Stefanyshyn from PyCharm]

Then, I wrapped my commands in a façade, which is the "Calculator" class and the commands tell it what to do. This is known as the command pattern.

If I need to change anything about how my object is returned, I can go to my create method and change it there, so then everywhere else in the program will have those changes.

This relates back to the factory pattern. The create function returns a new instance of an object.
