AAA Testing with Python

What is AAA Testing?

AAA Testing, also known as Arrange-Act-Assert,is a great way to structure test cases. It describes the order of operations that need to be performed to write a good test. AAA Testing represents a recognizable test template following the A-A-A pattern of testing. The pattern focuses each test on a single action. By applying AAA, you can ensure that your test is clear and more understandable and manageable.

For more information on AAA Testing, please read the article Arrange-Act-Assert

The A-A-A Pattern (Arrange-Act-Assert):
  1. Arrange: inputs and targets. Arrange steps sets up the test case. It will tell you if the test requires any objects or special settings, if it needs to log into a web app, etc.
  2. Act: acts on the target behavior. Act steps covers the main thing to be tested, like calling a function or method.
  3. Assert: asserts expected outcomes. Act steps returns some sort of a response and Assert steps verify the “goodness or badness” of that response. This step determines if the test PASSES or FAILS. Sometimes, assertions can be simple, like checking if a calculator adds, subtracts, multiplies or divides numbers properly. Other times, assertions can require checking multiple facets of a system.

Every programming language has testing frameworks. I will discuss Pytest, as one of the most popular frameworks.

Unit Testing:

Unit Testing is a method that is used to test programs and applications to see if they function correctly and without any problems. It is part of software testing, which involves many steps. If any problems occur, they should be taken care of early on, so that they don't show up later for the end users.

The main purpose behind unit testing is to check that all individual parts are working well like it is supposed to. The entire system will only be able to work well if the individual parts are working well.

There are two types of unit testing: manual and automated. The automated method is most preferred because it is faster and more accurate. Automated testing usually involved the developer first writing a section of the code in the application so the function can be tested. The manual approach has a step-by-step instructional procedure that helps developers perform the test.

Here is an example of a basic unit test for Python’s multiplication of a calculator.

  1. The Arrange Step creates a variable named “calc” for testing. It instantiates the calc class.
  2. The Act Step calls the “multiplying_numbers” function using the “calc” variable and stores the returned variable named “result”.
  3. The Assert Step verifies that “result” is correct.
